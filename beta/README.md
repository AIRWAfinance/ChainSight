# Beta launch kit

Practical tools for landing the first 5-10 ChainSight beta users.

| File | What it's for |
|------|----------------|
| [`OUTREACH.md`](OUTREACH.md) | Email + DM templates. Cold outreach + follow-up cadence. |
| [`FEEDBACK_QUESTIONS.md`](FEEDBACK_QUESTIONS.md) | The 19 questions to ask each beta user at week 2. |
| [`IP_NOTES.md`](IP_NOTES.md) | Bet365 employment-contract sanity check before outreach. |

## Workflow

1. **Read [`IP_NOTES.md`](IP_NOTES.md) first.** Don't email anyone connected
   to your day job.
2. List 30 names worth reaching out to — friends-of-friends, LinkedIn
   connections, ex-colleagues at non-betting companies. Aim for compliance,
   crypto, or fintech roles.
3. Send 5 outreach messages a day for a week using
   [`OUTREACH.md`](OUTREACH.md). Track replies in a simple spreadsheet.
4. The moment someone says yes, send a Loom walking through the product.
5. Schedule the 2-week follow-up with the
   [`FEEDBACK_QUESTIONS.md`](FEEDBACK_QUESTIONS.md) — the answers tell you
   what to build next.

## Realistic targets

- **Week 1:** 30 outreach messages sent.
- **Week 2:** 3-5 demo calls booked.
- **Week 4:** 2-3 active beta users actually using the product weekly.
- **Week 8:** 1-2 of those willing to be a paying customer with the right
  pricing.

If after 4 weeks of consistent outreach you have 0 active users, the
problem is the **pitch** or the **target list**, not the product.
Re-read those messages, change one variable at a time, retry.
